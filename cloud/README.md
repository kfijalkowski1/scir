# SCIR Cloud Infrastructure

This directory contains the OpenTofu + Terragrunt implementation for the washing machine IoT monitor.

## Toolchain

- OpenTofu language compatibility: Terraform `1.14`
- AWS provider: `~> 6.0`
- Terragrunt: latest (v1.x)

## Layout

- `root.hcl` - global Terragrunt configuration (provider generation and S3 remote state)
- `environments/prod/*` - Terragrunt units and dependency graph
- `modules/*` - reusable Terraform modules

## Before first apply

0. Create an IAM user with `AdministratorAccess` IAM policy attached, create security credentials and add them as a profile to your local AWS CLI. This setup will **not** work using the root IAM user!

1. Export required environment variables:

```bash
export SCIR_DISCORD_WEBHOOK_URL="https://discord.com/api/webhooks/..."
export SCIR_WEBHOOK_AUTH_TOKEN="<long-random-token>"
```

2. Move into the environment folder:

```bash
cd environments/prod
```

3. Initialize and apply in dependency order using Terragrunt:

```bash
terragrunt run --all --backend-bootstrap --working-dir . plan
terragrunt run --all --backend-bootstrap --working-dir . apply
```

## Notes

- MQTT topics are generated as:
  - Telemetry filter: `scir/prod/washer/+/status/switch:0`
  - Control/events topic: `scir/prod/washer/buzzer/events`
- IoT certificates are generated by Terraform in this implementation.
  - Private keys will be stored in state; protect state access strictly.
- CloudWatch dashboard resource is managed by Terraform.
  - Public share link setup may still require a manual AWS Console action.
